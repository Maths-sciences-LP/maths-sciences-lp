Chapitre 25 – Algorithmique appliquée

BTS | Mathématiques | Groupements D1, D2

Objectifs du chapitre
  • Rappeler les structures de base d'un algorithme (variables, conditions, boucles).
  • Maîtriser les structures de données : tableaux, listes, dictionnaires.
  • Comprendre et comparer les algorithmes de tri (sélection, insertion, rapide).
  • Implémenter les algorithmes de recherche (linéaire, dichotomique).
  • Comprendre la récursivité et analyser la pile d'appels.
  • Appliquer les méthodes numériques (Newton, trapèzes, Euler) à des problèmes scientifiques.
  • Évaluer la complexité algorithmique et choisir l'algorithme approprié.
Situation professionnelle — Automatisation d'une analyse en laboratoire de biologie

Un technicien de laboratoire en biologie médicale traite quotidiennement des séries de plusieurs centaines de résultats de dosages (concentrations en glucose, créatinine, cholestérol…). Pour accélérer l'analyse, il programme des scripts qui automatisent : le tri des résultats par valeur, la recherche d'un seuil critique, le calcul de la concentration moyenne par méthode des trapèzes (pour une courbe d'étalonnage), et la détection des valeurs aberrantes. Cet outillage algorithmique lui fait gagner plusieurs heures de travail par semaine tout en réduisant les erreurs de saisie.

1. Rappels d'algorithmique

Définition — Algorithme

Un algorithme est une suite finie d'instructions non ambiguës permettant de résoudre un problème en un nombre fini d'étapes. Tout algorithme doit être :

  • Correct : produit le résultat attendu pour toutes les entrées valides.
  • Terminant : se termine en un nombre fini d'étapes.
  • Efficace : utilise raisonnablement les ressources (temps, mémoire).
Méthode — Structures de base en pseudo-code

Variable et affectation :

Variable x : Entier
x ← 5           // affectation
x ← x + 1       // incrémentation

Condition :

Si x > 0 Alors
    Afficher("positif")
Sinon Si x = 0 Alors
    Afficher("nul")
Sinon
    Afficher("négatif")
Fin Si

Boucle Tant que :

Tant que x > 0 Faire
    x ← x - 1
Fin Tant que

Boucle Pour :

Pour i allant de 1 à n Faire
    Afficher(T[i])
Fin Pour

2. Structures de données

Définition — Tableau (Array)

Un tableau est une collection ordonnée d'éléments de même type, accessibles par un indice (entier).

  • Accès à un élément : \(O(1)\) (temps constant).
  • Insertion/suppression au milieu : \(O(n)\) (déplacement des éléments).
  • Taille fixe en mémoire statique (ou dynamique selon le langage).
T ← [12, 4, 7, 19, 3]  // tableau de 5 éléments
T[0] = 12              // premier élément (indice 0 en Python)
T[4] = 3               // dernier élément
Définition — Liste chaînée

Une liste chaînée est une structure où chaque élément (nœud) contient une valeur et un pointeur vers l'élément suivant.

  • Insertion/suppression en tête : \(O(1)\).
  • Accès par indice : \(O(n)\) (parcours séquentiel).
  • Usage : piles, files, listes à taille variable.
Définition — Dictionnaire (Hashmap)

Un dictionnaire associe des clés à des valeurs (paires clé : valeur). L'accès se fait par la clé, pas par un indice.

  • Accès, insertion, suppression : \(O(1)\) en moyenne (table de hachage).
  • Les clés doivent être uniques et immuables.
mesure ← {"glucose": 1.2, "créatinine": 85, "cholestérol": 4.8}
mesure["glucose"] = 1.2    // accès par clé
Propriétés — Comparaison des structures
StructureAccèsInsertionSuppressionUsage principal
Tableau\(O(1)\)\(O(n)\)\(O(n)\)Données ordonnées, tri
Liste chaînée\(O(n)\)\(O(1)\)*\(O(1)\)*Piles, files

3. Algorithmes de tri

Définition — Problème du tri

Étant donné un tableau de \(n\) éléments comparables, le tri consiste à les réordonner dans un ordre donné (croissant ou décroissant). Les algorithmes de tri se comparent par leur complexité temporelle (nombre d'opérations) et leur complexité en espace mémoire.

3.1 Tri par sélection

Méthode — Tri par sélection

Principe : à chaque étape, sélectionner le minimum du sous-tableau non trié et l'échanger avec le premier élément non trié.

Procédure TriSelection(T, n)
  Pour i de 0 à n-2 Faire
    iMin ← i
    Pour j de i+1 à n-1 Faire
      Si T[j] < T[iMin] Alors iMin ← j
    Fin Pour
    Échanger(T[i], T[iMin])
  Fin Pour
Fin Procédure

Complexité : \(O(n^2)\) dans tous les cas. Stable : Non. En place : Oui.

Exemple — Tri par sélection sur [64, 25, 12, 22, 11]
ÉtapeTableauAction
0[64, 25, 12, 22, 11]Min = 11 (pos 4) → échange 64 et 11
1[11, 25, 12, 22, 64]Min = 12 (pos 2) → échange 25 et 12
2[11, 12, 25, 22, 64]Min = 22 (pos 3) → échange 25 et 22
3[11, 12, 22, 25, 64]Min = 25 (pos 3) → pas d'échange
Fin[11, 12, 22, 25, 64]Tableau trié

3.2 Tri par insertion

Méthode — Tri par insertion

Principe : à chaque étape, insérer le prochain élément à sa bonne position dans la partie déjà triée, en décalant les éléments plus grands.

Procédure TriInsertion(T, n)
  Pour i de 1 à n-1 Faire
    clé ← T[i]
    j ← i - 1
    Tant que j ≥ 0 et T[j] > clé Faire
      T[j+1] ← T[j]
      j ← j - 1
    Fin Tant que
    T[j+1] ← clé
  Fin Pour
Fin Procédure

Complexité : \(O(n^2)\) dans le pire cas, \(O(n)\) si déjà trié. Stable : Oui. Très efficace pour les petits tableaux.

3.3 Tri rapide (Quicksort)

Méthode — Tri rapide (Quicksort)

Principe (diviser pour régner) :

  1. Choisir un pivot dans le tableau.
  2. Partitionner le tableau : éléments ≤ pivot à gauche, > pivot à droite.
  3. Appliquer récursivement le tri sur chaque sous-tableau.
Fonction TriRapide(T, g, d)   // tri de T[g..d]
  Si g < d Alors
    p ← Partition(T, g, d)    // p = position finale du pivot
    TriRapide(T, g, p-1)
    TriRapide(T, p+1, d)
  Fin Si
Fin Fonction

Complexité : \(O(n \log n)\) en moyenne, \(O(n^2)\) dans le pire cas (pivot toujours extrêmal). En place : Oui (peu de mémoire supplémentaire). Très efficace en pratique.

4. Algorithmes de recherche

4.1 Recherche linéaire (séquentielle)

Méthode — Recherche linéaire

Parcourir le tableau élément par élément jusqu'à trouver la valeur cherchée.

Fonction RechercheLinéaire(T, n, val) : Entier
  Pour i de 0 à n-1 Faire
    Si T[i] = val Alors Retourner i
  Fin Pour
  Retourner -1   // valeur non trouvée
Fin Fonction

Complexité : \(O(n)\). Fonctionne sur tout tableau (trié ou non).

4.2 Recherche dichotomique (binaire)

Méthode — Recherche dichotomique

Applicable uniquement sur un tableau trié. À chaque étape, on compare la valeur cherchée au milieu et on élimine la moitié du tableau.

Fonction RechercheDicho(T, val) : Entier
  g ← 0 ; d ← Longueur(T) - 1
  Tant que g ≤ d Faire
    m ← (g + d) div 2       // milieu
    Si T[m] = val Alors Retourner m
    Sinon Si T[m] < val Alors g ← m + 1
    Sinon d ← m - 1
  Fin Tant que
  Retourner -1
Fin Fonction

Complexité : \(O(\log_2 n)\). Pour 1 million d'éléments, au plus 20 comparaisons !

5. Récursivité

Définition — Fonction récursive

Une fonction récursive est une fonction qui s'appelle elle-même dans sa définition. Elle doit toujours comporter :

  • Un ou plusieurs cas de base (terminaison, sans appel récursif).
  • Un ou plusieurs cas récursifs qui réduisent le problème vers le cas de base.

5.1 Factorielle

Exemple — Factorielle récursive $$n! = \begin{cases} 1 & \text{si } n = 0 \\ n \times (n-1)! & \text{si } n \geq 1 \end{cases}$$ 
Fonction Fact(n) : Entier
  Si n = 0 Alors Retourner 1       // cas de base
  Sinon Retourner n × Fact(n-1)    // cas récursif
Fin Fonction

Appel Fact(4) :

Fact(4) = 4 × Fact(3)
Fact(3) = 3 × Fact(2)
Fact(2) = 2 × Fact(1)
Fact(1) = 1 × Fact(0)
Fact(0) = 1 ← cas de base

Résultat : \(4! = 4 \times 3 \times 2 \times 1 = 24\).

5.2 Suite de Fibonacci

Exemple — Fibonacci récursif $$F(n) = \begin{cases} 0 & \text{si } n=0 \\ 1 & \text{si } n=1 \\ F(n-1) + F(n-2) & \text{si } n \geq 2 \end{cases}$$ 
Fonction Fib(n) : Entier
  Si n ≤ 1 Alors Retourner n
  Sinon Retourner Fib(n-1) + Fib(n-2)
Fin Fonction

Problème : cette version naïve est très lente : \(O(2^n)\). De nombreux calculs sont répétés. Solution : utiliser la mémoïsation (stocker les résultats déjà calculés) ou une version itérative.

5.3 Tours de Hanoï

Exemple — Tours de Hanoï

Déplacer \(n\) disques de la tige A vers la tige C en utilisant la tige B, sans jamais poser un disque sur un plus petit.

Procédure Hanoï(n, source, cible, auxiliaire)
  Si n = 1 Alors
    Afficher("Déplacer disque 1 de " + source + " vers " + cible)
  Sinon
    Hanoï(n-1, source, auxiliaire, cible)
    Afficher("Déplacer disque " + n + " de " + source + " vers " + cible)
    Hanoï(n-1, auxiliaire, cible, source)
  Fin Si
Fin Procédure

Nombre de déplacements pour \(n\) disques : \(2^n - 1\). Complexité : \(O(2^n)\).

Attention — Pile d'appels et dépassement

Chaque appel récursif empile un cadre d'appel en mémoire. Une récursion trop profonde provoque un Stack Overflow (débordement de pile). En pratique, les fonctions récursives à grande profondeur doivent être converties en version itérative ou utiliser la récursion terminale.

6. Algorithmes numériques

6.1 Méthode de Newton — Zéro d'une fonction

Méthode de Newton (Newton-Raphson)

Calcule une valeur approchée de \(f(x) = 0\) en linéarisant itérativement la fonction.

Formule de récurrence :

$$x_{n+1} = x_n - \frac{f(x_n)}{f'(x_n)}$$

Algorithme :

Fonction Newton(f, fprime, x0, ε) : Réel
  x ← x0
  Tant que |f(x)| > ε Faire
    x ← x - f(x) / fprime(x)
  Fin Tant que
  Retourner x
Fin Fonction

Convergence quadratique (le nombre de décimales exactes double à chaque itération si \(f'(x^*) \neq 0\)).

Exemple — Newton pour \(\sqrt{2}\) (\(f(x) = x^2 - 2\))

\(f'(x) = 2x\). Formule : \(x_{n+1} = x_n - \frac{x_n^2 - 2}{2x_n} = \frac{x_n + 2/x_n}{2}\).

6.2 Méthode des trapèzes — Intégration numérique

Méthode des trapèzes

Approximation de l'intégrale \(\displaystyle\int_a^b f(x)\,dx\) par la somme d'aires de trapèzes :

$$\int_a^b f(x)\,dx \approx \frac{h}{2}\left[f(x_0) + 2f(x_1) + 2f(x_2) + \cdots + 2f(x_{n-1}) + f(x_n)\right]$$

où \(h = \dfrac{b-a}{n}\) et \(x_i = a + ih\).

Fonction Trapèzes(f, a, b, n) : Réel
  h ← (b - a) / n
  S ← f(a) + f(b)
  Pour i de 1 à n-1 Faire
    S ← S + 2 × f(a + i × h)
  Fin Pour
  Retourner S × h / 2
Fin Fonction

Erreur : \(O(h^2)\). Doubler \(n\) divise l'erreur par 4.

Exemple — Calcul de l'aire sous une courbe d'étalonnage

En laboratoire, la courbe d'absorption d'un réactif entre 0 et 4 s est approximée par \(f(t) = 0.5t^2 - t + 2\). Calculer \(\displaystyle\int_0^4 f(t)\,dt\) avec \(n=4\) trapèzes.

\(h = 1\). Valeurs : \(f(0)=2,\; f(1)=1.5,\; f(2)=2,\; f(3)=3.5,\; f(4)=6\).

$$I \approx \frac{1}{2}\left[2 + 2(1.5) + 2(2) + 2(3.5) + 6\right] = \frac{1}{2}[2+3+4+7+6] = \frac{22}{2} = 11$$

Valeur exacte : \(\left[\frac{t^3}{6}-\frac{t^2}{2}+2t\right]_0^4 = \frac{64}{6}-8+8 = \frac{32}{3} \approx 10{,}67\). Erreur ≈ 3%.

6.3 Méthode d'Euler — Équations différentielles

Méthode d'Euler

Résolution numérique approchée de \(y' = f(t, y),\; y(t_0) = y_0\).

$$y_{n+1} = y_n + h \cdot f(t_n, y_n) \qquad t_{n+1} = t_n + h$$ 
Procédure Euler(f, t0, y0, T, h)
  t ← t0 ; y ← y0
  Tant que t < T Faire
    y ← y + h × f(t, y)
    t ← t + h
    Afficher(t, y)
  Fin Tant que
Fin Procédure

La méthode d'Euler explicite est simple mais peu précise (\(O(h)\) d'erreur). Des méthodes plus précises (Runge-Kutta d'ordre 4) sont utilisées en pratique.

Exemple — Cinétique de dégradation d'un médicament

La concentration \(C(t)\) d'un médicament suit \(C' = -k\,C\) avec \(k = 0.2\,\text{h}^{-1}\) et \(C(0) = 100\,\text{mg/L}\). Approche numérique avec \(h = 1\,\text{h}\) :

$$C_{n+1} = C_n + h \cdot (-k\,C_n) = C_n(1 - kh) = 0.8\,C_n$$

7. Complexité algorithmique

Définition — Notation O (Grand O)

La complexité algorithmique mesure la croissance du nombre d'opérations en fonction de la taille \(n\) de l'entrée. On utilise la notation \(O\) (grand O) pour exprimer le comportement asymptotique.

\(f(n) = O(g(n))\) signifie qu'il existe des constantes \(C > 0\) et \(n_0\) telles que \(f(n) \leq C\cdot g(n)\) pour tout \(n \geq n_0\).

8. Applications D1/D2 — Traitement de données biologiques

Exemple D1 — Tri et analyse d'une série de mesures de glycémie

Un technicien de laboratoire dispose de 200 résultats de glycémie (en g/L). Il souhaite :

  1. Trier les résultats (tri rapide pour \(n=200\) : très rapide).
  2. Trouver rapidement si un résultat dépasse le seuil 2,0 g/L (recherche dichotomique après tri).
  3. Calculer la valeur moyenne et l'écart-type.
Fonction AnalyseGlycémie(mesures, n) : Réel, Réel
  TriRapide(mesures, 0, n-1)
  somme ← 0
  Pour i de 0 à n-1 Faire
    somme ← somme + mesures[i]
  Fin Pour
  moy ← somme / n
  sommeCarrés ← 0
  Pour i de 0 à n-1 Faire
    sommeCarrés ← sommeCarrés + (mesures[i] - moy)²
  Fin Pour
  écartType ← Racine(sommeCarrés / n)
  Retourner moy, écartType
Fin Fonction

Complexité totale : \(O(n\log n)\) pour le tri + \(O(n)\) pour la moyenne = \(O(n\log n)\).

Exemple D2 — Calcul numérique de concentration par intégration

Pour déterminer la quantité totale d'un médicament absorbée par un patient sur 8 heures, le technicien dispose de mesures de concentration à intervalles réguliers (toutes les heures). L'intégrale \(\displaystyle\int_0^8 C(t)\,dt\) donne l'AUC (Area Under the Curve), paramètre pharmacocinétique clé.

Fonction AUC_Trapèzes(C, t) : Réel
  // C et t sont des tableaux de même taille n
  n ← Longueur(C)
  AUC ← 0
  Pour i de 0 à n-2 Faire
    AUC ← AUC + (C[i] + C[i+1]) × (t[i+1] - t[i]) / 2
  Fin Pour
  Retourner AUC   // en mg·h/L
Fin Fonction

Synthèse du chapitre 25
  • Structures de données : tableau \(O(1)\) accès, liste chaînée \(O(1)\) insertion tête, dictionnaire \(O(1)\) accès clé.
  • Tris : sélection et insertion en \(O(n^2)\) ; tri rapide en \(O(n\log n)\) en moyenne.
  • Recherche : linéaire \(O(n)\) ; dichotomique \(O(\log n)\) sur tableau trié.
  • Récursivité : toujours un cas de base + cas récursif réduisant le problème.
  • Newton : \(x_{n+1} = x_n - f(x_n)/f'(x_n)\). Trapèzes : \(h/2 \times [f(x_0)+2f(x_1)+\cdots+f(x_n)]\). Euler : \(y_{n+1}=y_n+hf(t_n,y_n)\).
  • Complexité : \(O(1) \ll O(\log n) \ll O(n) \ll O(n\log n) \ll O(n^2) \ll O(2^n)\).
